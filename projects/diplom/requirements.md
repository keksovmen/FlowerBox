# Сервер HTTP не HTTPS
1. необходим **URL** по которому буду обращаться к серверу
2. **GET** запрос данного **URL** возвращает статус **200 либо 404**
	- 200 если данные есть, устройство после выполнения сразу же запросит следующие
	- 404 данных нет, устройство подождет условное время до следующего запроса

# Передаваемые данные
Это данные, которые устройство считает при **GET** запросе и 200 статус код

Это один объект **JSON** следующей структуры:

```
{
	"id" : int,
	"value" : "string"
}
```

- **id** - это уникальный ИД который хотим изменить, 
	смотри таблицу ниже, должен быть целочисленным
- **value** - это желаемое значение, **не зависимости от типа данных, должно быть превращено в СТРОКУ**

## Таблица ИД и значений
Пока временное решение и ИД могут измениться так же как и значения
Так же будут добавляться новые ИД

|  id  | описание | тип данных | value |
| ---- | -------- | ---------- | ----- |
| 0    | включение выключение фонаря		  | int | 1 - вкл, 2 - выкл  |
| 1    | цвет фонаря сразу все каналы		  | int | число с бинарным форматом 0xWWRRGGBB - где WW белый цвет, RR красный, GG зеленый, BB синий, значения от 0 до 255 на цвет |
| 2    | воспроизведение трека				  | int | индекс трека от 0 до количества треков на карте |
| 3    | выключение проигрывания трека 		  | int | 1 - выключить |
| 4    | громкость воспроизведения			  | int | значения от 0 до 30 включительно, 0 = тихо, 30 = громко |

### Пример про цвет фонаря
Хотим допустим яркий (255) красный, получается надо запихать в бинарный формат следующее:
*0x00FF0000 что будет равно числу 16711680* и конечный JSON будет выглядить так

```
{
	"id" : 1,
	"value" : "16711680"
}
```
